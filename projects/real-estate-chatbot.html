Project Overview

An intelligent real estate chatbot system that leverages Retrieval-Augmented Generation (RAG) to help users find apartments and properties in Buenos Aires. The system combines natural language processing with sophisticated database queries and filtering capabilities to provide accurate, contextual responses to real estate inquiries.

Built with a production-ready architecture using FastAPI, PostgreSQL, and Docker, this chatbot demonstrates advanced AI engineering practices in a real-world application domain.

Advanced RAG Implementation

Pre-filtering Stage

Initial filtering based on explicit user requirements such as location, price range, and property type before vector search to improve efficiency and relevance.

Vector Search

Semantic similarity search using embeddings to find properties that match the user's intent, even when expressed in natural language.

Post-filtering

Additional filtering applied after vector search to ensure results meet specific criteria that may not be captured in the embeddings.

Reranking

Final reordering of results using advanced algorithms to present the most relevant properties first, considering multiple factors like user preferences, property quality, and availability.

Data Integration

Airbnb Data Processing

The system processes real Airbnb listing data from Buenos Aires, including:

  • Property Details: Location, size, amenities, and descriptions
  • Pricing Information: Daily rates, seasonal variations, and additional fees
  • Availability: Calendar data and booking restrictions
  • Host Information: Host details and response rates
  • Reviews: Guest feedback and ratings

Vector Database Creation

Property descriptions and features are converted into high-dimensional vectors using state-of-the-art embedding models, enabling semantic search capabilities that understand context and intent rather than just keyword matching.

Technical Challenges & Solutions

Challenge: Handling ambiguous user queries while maintaining search accuracy.

Solution: Implemented multi-stage RAG pipeline with pre-filtering and reranking to balance broad search with precise results.

Challenge: Scaling vector search operations for large property datasets.

Solution: Optimized PostgreSQL with vector extensions and implemented efficient indexing strategies.

Challenge: Maintaining conversation context across multiple interactions.

Solution: Designed stateful conversation management with context preservation and intelligent memory management.
